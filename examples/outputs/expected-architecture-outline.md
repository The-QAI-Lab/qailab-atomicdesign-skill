# Expected architecture outline

A strong response should resemble:

## Product Scope
- audience
- goals
- platform
- page/screen inventory
- constraints

## Assumptions
- design assumptions
- responsive assumptions
- technology assumptions

## Atoms
- tokens
- primitive controls
- state primitives

## Molecules
- compact reusable combinations

## Organisms
- reusable feature sections

## Templates
- dashboard template
- content page template
- detail template
- settings template

## Pages / Screens
- route-level or screen-level final surfaces

## Implementation Mapping
- SwiftUI or React folder/component mapping
