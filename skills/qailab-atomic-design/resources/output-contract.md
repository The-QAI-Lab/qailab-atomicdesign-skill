# Output contract

When the skill is active, the agent response should usually contain these sections in order:

1. Product Scope
2. Assumptions
3. Atoms
4. Molecules
5. Organisms
6. Templates
7. Pages / Screens
8. Reuse Rules
9. Implementation Mapping
10. Next Build Step

## Minimum quality bar

A good response:

- maps all major UI elements to the hierarchy
- includes reusable design primitives
- defines component states
- distinguishes templates from pages
- explains platform behavior
- avoids implementation drift

## Optional add-ons

When useful, also include:

- folder structure
- token naming system
- accessibility considerations
- responsive breakpoints
- design review checklist
- code scaffold plan

## Failure indicators

The response is weak if it:

- jumps straight to pages
- lists only pages with no components
- has no atoms or tokens
- mixes organisms and templates together
- ignores state modeling
